## 1. トークノミクス詳細分析

### 1.1 トークン供給構造

| 項目 | 数値 |
|------|------|
| 総発行枚数 | 1,000,000,000 SANAET (10億枚) |
| 流通供給量 | 事実上10億枚すべて（ロック無し） |
| 新規発行 | 2026年3月4日に停止 |
| バーン機能 | 公式記録なし |

#### 🔴 重大な問題点：「即時全量流通モデル」
オンチェーン分析（Solscan）によると、**ベスティングコントラクトが存在せず**、発行と同時に全供給量が事実上流通可能な状態に置かれています。これは標準的なトークン設計と比較して著しく問題があります。

```
発表されたアロケーション:

  │  チーム        :  5%  (0.5億枚)         │
  └─────────────────────────────────────────┘
```

#### アロケーション構造の問題点（詳細）

**① エコシステム枠（65%）の不透明性**
- 「エコシステム」という曖昧な名目でトークンの3分の2を運営側が保持
- ロック・ベスティングの記録がオンチェーン上に存在しない
- 実態は**運営側のリザーブとして機能**しており、いつでも市場放出が可能
- 健全なトークン設計では、エコシステム枠にも通常2〜4年のベスティングが設定される

**② コミュニティ枠（20%）のエアドロップ設計の欠陥**
- エアドロップは話題性でユーザーを集める手法として有効だが、本プロジェクトでは**配布基準・スナップショット日時・配布条件が非公開**
- 不公平配布によるインサイダー利益の可能性を排除できない

**③ チーム枠（5%）のベスティング不在**
- 通常プロジェクトでは、チーム枠には1年クリフ＋3〜4年リニア解除が標準
- SANAETではチーム枠5,000万枚が即座に利用可能な状態と推定される
- ただし「65%エコシステム」が実質チーム保有であれば、名目上の5%は過小表示

### 1.4 インフレ・デフレメカニズム

**インフレ要因**
- 発行上限は設定されているが、当初より**全量が流通可能**な状態（実質的即時インフレ）
- エコシステム枠（65%）の放出タイミングは運営裁量に依存 → 売り圧力の制御不能

**デフレメカニズム**
- トークンバーン機能：**公式確認なし**
- 取引手数料によるバーン：**確認されていない**
- ステーキングによる供給減少：**仕組みなし**
- 2026年3月4日の新規発行停止は「デフレ措置」ではなく**プロジェクト終了の実質宣言**

> **⚠️ 評価**: SANAETには意味のあるデフレメカニズムが存在せず、供給サイドからの価値支持が全くない。
